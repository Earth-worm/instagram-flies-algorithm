# instagram-flies-algorithm
## 名前
インスタ蝿アルゴリズム

## 着想
人間がSNSでいいねを獲得しようとする行動を模した進化的計算手法
個体:インスタグラマー、youtuber
評価:いいね数

## 流れ
- ランダムで人間を初期化
- いいね数を計算
- 個体をクラスタリング
- カテゴリの中で優秀な個体に近づいて位置を更新
- 伸びが少ない個体は他のカテゴリと組み合わせて位置を更新させる
- 急上昇カテゴリに乗っかりたい
- 自分の軸となるカテゴリを決める
- いいね数計算に戻る

## クラスタリング手法
当面はkmeansを使用
理由
- 自動クラスタ数決定のアルゴリズムだとクラスタ数が変動してしまう。次の戦略を決定する際に今のクラスタにどれだけ属すかも反映させたい。
- クラスタリング時に前の反復で計算された重心を使用することで計算量が少なくなる
候補
 - エルボー法(計算量が少ない、クラスタ内誤差平方和を最小化する手法)
 - DBSCAN(計算量が多い、複雑なクラスタリングを行うことができる)

## 位置の更新
できればマルコフ連鎖みたいなことをやりたい遷移先、専門を極める、トレンドを追う、開拓、これらの遷移確率を更新していって各々の最適な戦略を獲得して欲しい
遷移確率の更新は未決定

- 専門を極める
自分のbestポジションとクラスタ内のbestポジションから位置を更新する

- トレンドを追う
ルーレット選択で選択された他のクラスタを試してみる
ルーレト選択にはクラスタ内の最も優秀な個体のいいね数を参考にする(or 平均)

- 開拓
現在のクラスタから全く異なる場所に遷移し、新たなクラスタ作成の手がかりとなる
位置の更新には適当な乱数分布を使用する(できたらピークが1にきて、クラスタごとの平均距離をスケールにして、クラスタごとの距離がいい感じなってくれたら嬉しい)

## 懸念点
クラスタリング手法に自動でクラスタ数を決定する手法を採用すると計算量が不安
DBSCANを使用するのであれば、位置を更新するためにクラスタ重心を使用するのは間違っているかもしれない
トレンドを追うでは新たに現れた開拓者が選択されるようにしたい。そのためトレンドを作成する必要がありそう
